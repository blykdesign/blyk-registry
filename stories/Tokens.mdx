# Design Tokens

blyk uses CSS variables as design tokens, following the same convention as [shadcn/ui theming](https://ui.shadcn.com/docs/theming). All tokens are defined in `registry/base/tokens/` and mapped to Tailwind CSS via the `@theme inline` block.

## Convention

Colors follow a **background / foreground** convention. Each semantic color has a pair:

```css
--primary: 0 0% 9%;            /* Background */
--primary-foreground: 0 0% 98%; /* Text on primary background */
```

This lets you use `bg-primary text-primary-foreground` for consistent contrast.

## Dark Mode

Dark mode is activated by adding the `dark` class to the root element:

```html
<html class="dark">
```

All color tokens automatically switch to their dark values.

## Custom Themes

To create a custom theme, override the CSS variables:

```css
[data-theme="custom"] {
  --primary: 220 90% 56%;
  --primary-foreground: 0 0% 100%;
  /* ... override other tokens */
}
```

The blyk registry supports multiple themes. See the `registry/base/tokens/` directory for the base token definitions.
